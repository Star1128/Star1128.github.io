一些问题

代码冲突的解决

如果你当前使用的代码不是最新版，那么当你企图Push到远程仓库的时候，会遭到拒绝

此时需要将远程仓库的最新版代码pull到本地，当然会提示代码冲突：

然后你需要手动合并代码，Git会提示你产生冲突的地方，以以下格式展现：

你要手动处理是保留一个还是保留全部，处理好之后再执行Add->Commit->Push即可

Merge和Rebase的区别

假设你将远程仓库的master分支拉到本地并有2次commit D、E，在此期间你的同事在master上提交了2次commit C、F

此时你想将本地master合并到远程仓库的master，会提示代码冲突

随后你在本地执行git pull origin master拉取最新代码，会在本地master分支上产生一次merge记录并产生新的commit G：

而如果你是在本地master执行git pull --rebase origin master拉取最新代码并变基，会自动将你的commit D、E追加在commit F的后面（当然如果出现代码冲突还是需要你手动解决）：

  • 所以，通过rebase合并的代码是看不到merge记录的

文件过大无法上传

HTTP文件大小限制，有一次无法上传大量文件（总大小7M）

解决办法：将remote地址由https变更为SSH地址

名称问题

注意有的叫main有的叫master…  2022年Github默认主分支叫main了

初始化仓库冲突

本地创建项目目录，Github创建远程仓库并且自动生成Readme.md，将本地目录和远程仓库关联起来之后无法push，提示冲突

解决办法：

  1. 要么先建立远程仓库，pull/clone到本地之后再在里面创建项目目录
  2. 或者是不要让Github自动生成Readme.md，自己在本地写一个，一并push上去
  3. 直接git push -f origin master强推，会把远程仓库的Readme.md覆盖掉（但是不太建议强推）
